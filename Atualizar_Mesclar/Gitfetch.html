git fetch Atualizar / Mesclar

O comando git fetch baixa as alterações do repositório remoto mas não as aplica automaticamente. Permite verificar o que mudou antes de integrar ao projeto local.

Quando usar

Use git fetch quando quiser ver o que mudou no GitHub antes de decidir se vai aplicar as alterações no seu projeto.

Buscar as atualizações remotas

Baixa as informações mas não altera seus arquivos:

git fetch

Ver o que chegou do remoto

Compara sua branch local com a remota após o fetch:

git log HEAD..origin/main --oneline

Aplicar as alterações depois

Se quiser aplicar o que foi baixado, use o merge:

git merge origin/main

Diferença entre fetch e pull

git fetch só baixa. git pull baixa e já aplica. O pull é equivalente a:

git fetch
git merge origin/main

Exemplo completo

git fetch
git log HEAD..origin/main --oneline
git merge origin/main
💡 Dica: Use git fetch em projetos com muitos colaboradores para inspecionar as mudanças antes de aplicá-las e evitar surpresas.
