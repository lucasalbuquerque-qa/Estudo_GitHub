git rebase Atualizar / Mesclar

← Voltar

O comando git rebase move os commits de uma branch para a ponta de outra, reescrevendo o histórico de forma linear. Diferente do merge, ele não cria um commit extra de junção.

Quando usar

Use git rebase quando quiser manter o histórico do projeto limpo e linear, sem commits de merge desnecessários.

Ir para a branch que será atualizada

Entre na branch que quer rebasear sobre a main:

git checkout minha-feature

Executar o rebase

Move os commits da branch atual para a ponta da main:

git rebase main

Resolver conflitos se houver

Se houver conflito, resolva o arquivo, adicione e continue:

git add .
git rebase --continue

Cancelar o rebase

Se quiser abortar e voltar ao estado anterior:

git rebase --abort

Exemplo completo

git checkout minha-feature
git rebase main
git checkout main
git merge minha-feature
git log --oneline
💡 Dica: Nunca use git rebase em branches que já foram enviadas ao GitHub e compartilhadas com outras pessoas, pois ele reescreve o histórico e pode causar conflitos para os outros.
