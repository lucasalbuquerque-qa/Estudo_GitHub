.gitkeep Dicas

O .gitkeep é uma convenção usada para versionar pastas vazias no Git. Por padrão, o Git não rastreia diretórios vazios — ele só rastreia arquivos. O .gitkeep resolve isso.

O problema

O Git ignora pastas vazias. Se você criar uma pasta uploads/ vazia no projeto e fizer commit, ela não vai para o GitHub:

# Criou a pasta
mkdir uploads/

# Tentou adicionar — Git ignora pastas vazias
git add uploads/
git status
# Resultado: nothing to commit

A solução — criar o .gitkeep

Coloque um arquivo vazio chamado .gitkeep dentro da pasta para que o Git a reconheça:

# Cria o arquivo vazio dentro da pasta
touch uploads/.gitkeep

# Agora o Git detecta a pasta
git add uploads/.gitkeep
git status
# Resultado: new file: uploads/.gitkeep

Commitar a pasta

Agora é só commitar normalmente:

git add uploads/.gitkeep
git commit -m "Adiciona pasta uploads ao projeto"

Ignorar o conteúdo mas manter a pasta

Muito comum em pastas de upload ou cache: manter a pasta no Git mas ignorar tudo dentro dela. Configure o .gitignore assim:

# No arquivo .gitignore:
uploads/*
!uploads/.gitkeep

Isso ignora tudo em uploads/ exceto o próprio .gitkeep.

Casos de uso comuns

Pastas que costumam precisar de .gitkeep:

uploads/     # arquivos enviados por usuários
logs/        # arquivos de log gerados
cache/       # arquivos de cache
tmp/         # arquivos temporários
storage/     # armazenamento local

Exemplo completo

# Cria as pastas necessárias
mkdir -p uploads logs tmp

# Adiciona .gitkeep em cada uma
touch uploads/.gitkeep logs/.gitkeep tmp/.gitkeep

# Configura o .gitignore para ignorar o conteúdo
echo "uploads/*" >> .gitignore
echo "!uploads/.gitkeep" >> .gitignore

# Commita
git add .
git commit -m "Adiciona estrutura de pastas do projeto"
💡 Dica: O nome .gitkeep é apenas uma convenção — o arquivo pode ter qualquer nome. Mas .gitkeep é o mais reconhecido pela comunidade e deixa clara a intenção.
