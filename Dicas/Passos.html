Passo a Passo Git Dicas

Todos os fluxos principais do Git em uma página só. Do clone até o rebase — consulte o que precisar rapidamente.

📥 Clonar 🚀 Add→Commit→Push ⬇️ Atualizar local 🌿 Criar branch 🔗 Merge 🔁 Rebase ↩️ Desfazer commit 🤝 Fork e PR
1 Clonar repositório na máquina
01

Copie a URL do repositório no GitHub e clone:

git clone https://usuario@github.com/usuario/repo.git
02

Entre na pasta e verifique o estado:

cd nome-do-repo
git remote -v
git branch
git status
03

Configure o usuário se necessário (só para este projeto):

git config user.name "Seu Nome"
git config user.email "seu@email.com"
↓
2 Enviar alterações — add, commit e push
01

Veja o que foi alterado e revise as mudanças:

git status
git diff
02

Adicione os arquivos ao stage:

# Todos os arquivos
git add .

# Ou um específico
git add nome-do-arquivo.html
03

Crie o commit com uma mensagem clara:

git commit -m "feat: adiciona validação no formulário"
04

Envie para o GitHub:

git push
05

Confirme que foi enviado:

git status
git log --oneline -5
↓
3 Atualizar repositório local
01

Se tiver trabalho em andamento, guarde antes:

git stash save "trabalho em andamento"
02

Baixe e aplique as atualizações do GitHub:

git pull origin main
03

Restaure o trabalho que estava em andamento (se usou stash):

git stash pop
04

Se houver conflito, resolva os arquivos e finalize:

git add .
git commit -m "Resolve conflito após pull"
↓
4 Criar branch, alterar e enviar ao remoto
01

Parta sempre de uma main atualizada:

git switch main
git pull origin main
02

Crie a branch e mude para ela:

git switch -c feature/nome-da-feature
03

Faça as alterações e commite:

git add .
git commit -m "feat: descrição do que foi feito"
04

Envie a branch para o GitHub (primeiro push usa -u):

git push -u origin feature/nome-da-feature
↓
5 Merge da branch com a branch principal
01

Volte para a main e atualize:

git switch main
git pull origin main
02

Faça o merge da branch na main:

git merge feature/nome-da-feature
03

Se houver conflito, resolva e finalize:

git add .
git commit -m "Merge da feature/nome com a main"
04

Envie a main atualizada e delete a branch:

git push origin main
git branch -d feature/nome-da-feature
git push origin --delete feature/nome-da-feature
↓
6 Rebase da principal para a branch
01

Atualize a main local:

git switch main
git pull origin main
02

Volte para a branch e execute o rebase:

git switch feature/nome-da-feature
git rebase main
03

Se houver conflito, resolva e continue:

git add .
git rebase --continue

# Para cancelar:
git rebase --abort
04

Envie com force-with-lease (necessário após rebase):

git push --force-with-lease origin feature/nome-da-feature
↓
7 Desfazer commit
A

Corrigir a mensagem do último commit (antes do push):

git commit --amend -m "Mensagem corrigida"
B

Desfazer commit local mantendo as alterações:

git reset --soft HEAD~1
C

Desfazer commit local descartando tudo (cuidado — irreversível):

git reset --hard HEAD~1
D

Desfazer commit já enviado ao GitHub (seguro — cria novo commit):

git revert abc1234 --no-edit
git push
E

Recuperar commit deletado por engano com reflog:

git reflog
git checkout -b recuperado abc1234
Regra de ouro: se o commit já foi enviado ao GitHub, use git revert. Se ainda é só local, pode usar git reset.
↓
8 Contribuir com Fork e Pull Request
01

Faça o fork no GitHub e clone o seu fork:

git clone https://seu-usuario@github.com/seu-usuario/repo.git
cd repo
02

Conecte ao repositório original:

git remote add upstream https://github.com/dono-original/repo.git
03

Crie uma branch, faça as alterações e commite:

git switch -c fix/minha-correcao
git add .
git commit -m "fix: descreve a correção"
04

Atualize com o repositório original antes de enviar:

git fetch upstream
git rebase upstream/main
05

Envie para o seu fork e abra o Pull Request no GitHub:

git push -u origin fix/minha-correcao

Resumo de todos os fluxos

# 1. Clonar
git clone https://usuario@github.com/usuario/repo.git && cd repo

# 2. Enviar alterações
git add . && git commit -m "mensagem" && git push

# 3. Atualizar local
git stash && git pull origin main && git stash pop

# 4. Criar branch
git switch main && git pull && git switch -c feature/nome
git add . && git commit -m "feat: ..." && git push -u origin feature/nome

# 5. Merge
git switch main && git pull && git merge feature/nome && git push

# 6. Rebase
git switch main && git pull
git switch feature/nome && git rebase main
git push --force-with-lease origin feature/nome

# 7. Desfazer (local)
git reset --soft HEAD~1

# 7. Desfazer (já no GitHub)
git revert abc1234 --no-edit && git push
