Aliases do Git Dicas

Aliases são atalhos personalizados para comandos Git longos ou que você usa com frequência. Em vez de digitar git status, você pode criar git s e economizar tempo no dia a dia.

Como criar um alias

Use o git config com a opção --global para criar aliases que funcionam em todos os projetos:

git config --global alias.NOME "comando"

Aliases mais úteis do dia a dia

Crie atalhos para os comandos que você mais usa:

# Status resumido
git config --global alias.s "status -s"

# Log bonito em uma linha
git config --global alias.l "log --oneline --graph --decorate"

# Adicionar tudo
git config --global alias.aa "add ."

# Commit com mensagem
git config --global alias.c "commit -m"

# Push
git config --global alias.p "push"

# Trocar de branch
git config --global alias.sw "switch"

Usar os aliases criados

Depois de criar, é só usar normalmente:

# Em vez de: git status -s
git s

# Em vez de: git log --oneline --graph --decorate
git l

# Em vez de: git commit -m "mensagem"
git c "Adiciona nova funcionalidade"

Ver todos os aliases criados

Lista todos os aliases configurados:

git config --global --list | grep alias

Remover um alias

Para deletar um alias que não quer mais:

git config --global --unset alias.NOME

Exemplo completo — setup de aliases

git config --global alias.s "status -s"
git config --global alias.l "log --oneline --graph --decorate"
git config --global alias.aa "add ."
git config --global alias.c "commit -m"
git config --global alias.p "push"
git config --global alias.sw "switch"

# Usando no dia a dia:
git s
git aa
git c "Corrige bug no header"
git p
💡 Dica: Os aliases ficam salvos no arquivo ~/.gitconfig do seu computador. Você pode abrir e editar esse arquivo diretamente pelo VS Code para gerenciar todos de uma vez.
