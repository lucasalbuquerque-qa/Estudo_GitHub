Conventional Commits Dicas

O Conventional Commits é um padrão profissional de mensagens de commit que usa prefixos como feat:, fix: e docs: para deixar o histórico organizado, legível e até automatizável.

Estrutura da mensagem

O formato segue esse padrão:

tipo(escopo): descrição curta

# Exemplos:
feat(login): adiciona autenticação com Google
fix(carrinho): corrige cálculo de frete
docs(readme): atualiza instruções de instalação

Tipos principais

Cada prefixo tem um significado específico:

feat:     nova funcionalidade
fix:      correção de bug
docs:     mudança em documentação
style:    formatação, sem mudança de lógica
refactor: refatoração sem nova feature ou fix
test:     adição ou correção de testes
chore:    tarefas de manutenção, configs
perf:     melhoria de performance
ci:       mudanças em CI/CD
build:    mudanças no sistema de build

Com escopo (opcional)

O escopo identifica qual parte do projeto foi afetada:

feat(auth): adiciona login com GitHub
fix(api): corrige timeout nas requisições
docs(contributing): adiciona guia de contribuição
style(header): ajusta espaçamento do menu

Breaking changes

Para mudanças que quebram compatibilidade, adicione ! ou a nota no rodapé:

feat!: remove suporte ao Node 14

# Ou com rodapé:
feat(api): altera formato de resposta

BREAKING CHANGE: o campo "user" foi renomeado para "usuario"

Por que usar esse padrão

Com Conventional Commits você pode gerar changelogs automáticos, usar ferramentas como semantic-release para versionar automaticamente e ter um histórico que qualquer dev entende:

git log --oneline

# Resultado organizado:
feat(auth): adiciona login social
fix(cart): corrige bug no total
docs: atualiza README
chore: atualiza dependências

Exemplo completo

git commit -m "feat(cadastro): adiciona validação de CPF"
git commit -m "fix(login): corrige redirecionamento após logout"
git commit -m "docs: adiciona exemplos de uso na documentação"
git commit -m "chore: atualiza versão do Node no package.json"
git commit -m "refactor(api): simplifica lógica de autenticação"
