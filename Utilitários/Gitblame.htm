git blame Utilitários

O comando git blame mostra, linha por linha, quem foi o último a modificar cada parte de um arquivo, quando foi feito e em qual commit. Útil para entender a origem de um código.

Quando usar

Use git blame quando quiser saber quem escreveu uma linha específica de código, quando foi alterada, ou para investigar a origem de um bug.

Ver o blame de um arquivo

Mostra cada linha com o autor, data e hash do commit:

git blame nome-do-arquivo.html

Ver apenas um intervalo de linhas

Para focar em linhas específicas, use -L com o intervalo:

git blame -L 10,25 nome-do-arquivo.html

Ver com data simplificada

Exibe a data em formato mais curto e fácil de ler:

git blame --date=short nome-do-arquivo.html

Ignorar mudanças de espaço em branco

Para ignorar commits que só ajustaram indentação ou espaços:

git blame -w nome-do-arquivo.html

Exemplo completo

git blame index.html
git blame -L 5,20 index.html
git blame --date=short index.html
💡 Dica: No VS Code, a extensão GitLens mostra o blame diretamente no editor, ao lado de cada linha, sem precisar do terminal.
