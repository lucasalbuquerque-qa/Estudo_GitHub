git reflog Utilitários

O comando git reflog mostra o histórico completo de tudo que aconteceu no repositório local — incluindo commits deletados, resets e trocas de branch. É o "plano B" para recuperar trabalho perdido.

Quando usar

Use git reflog quando fizer um git reset --hard ou deletar commits por engano e precisar recuperá-los. O reflog guarda o histórico por 90 dias por padrão.

Ver o reflog completo

Lista todas as ações recentes com hash e descrição:

git reflog

Entendendo o resultado

Cada linha mostra o hash, posição (HEAD@{N}) e o que aconteceu:

abc1234 HEAD@{0}: commit: Adiciona login
def5678 HEAD@{1}: reset: moving to HEAD~1
ghi9012 HEAD@{2}: commit: Corrige bug no menu

Recuperar um commit perdido

Encontrou o hash do commit perdido no reflog? Restaure com:

git checkout abc1234

# Ou crie uma nova branch a partir dele:
git checkout -b recuperado abc1234

Desfazer um reset acidental

Se fez um git reset --hard por engano, use o reflog para voltar:

# Veja o reflog para achar o ponto anterior
git reflog

# Volte para aquele estado
git reset --hard HEAD@{2}

Exemplo completo — recuperar commit perdido

# Situação: fiz git reset --hard por engano

# 1. Abro o reflog para achar o commit
git reflog

# 2. Encontro o hash do commit perdido: abc1234

# 3. Crio uma branch para recuperar
git checkout -b recuperado abc1234

# 4. Verifico que o conteúdo voltou
git log --oneline
💡 Dica: O git reflog só funciona localmente — ele não existe no GitHub. É por isso que é tão importante não deletar sua pasta local antes de ter certeza que o remoto está atualizado.
