git cherry-pick Utilitários

O comando git cherry-pick aplica um commit específico de uma branch em outra, sem precisar fazer o merge completo. É como "colher" apenas um commit que você quer trazer.

Quando usar

Use git cherry-pick quando quiser trazer uma correção ou funcionalidade de outra branch sem mesclar tudo que está nela.

Encontrar o hash do commit

Primeiro veja o histórico da branch de origem para pegar o hash:

git log --oneline nome-da-branch

Ir para a branch de destino

Mude para a branch onde quer aplicar o commit:

git checkout main

Aplicar o commit

Use o hash do commit que quer trazer:

git cherry-pick abc1234

Aplicar vários commits de uma vez

Para trazer múltiplos commits, liste os hashes separados por espaço:

git cherry-pick abc1234 def5678 ghi9012

Resolver conflitos se houver

Se aparecer conflito, resolva os arquivos e continue:

git add .
git cherry-pick --continue

# Ou para cancelar:
git cherry-pick --abort

Exemplo completo

# Ver commits da outra branch
git log --oneline feature/pagamento

# Mudar para main
git checkout main

# Aplicar o commit específico
git cherry-pick abc1234

# Verificar resultado
git log --oneline
💡 Dica: O cherry-pick é muito usado para trazer hotfixes de uma branch de desenvolvimento para a branch de produção sem fazer merge de funcionalidades incompletas.
