git bisect Utilitários

O comando git bisect usa busca binária no histórico para encontrar exatamente qual commit introduziu um bug. Você diz quais commits são "bons" e "ruins" e o Git faz o trabalho de investigação automaticamente.

Quando usar

Use git bisect quando aparecer um bug e você não sabe em qual commit ele surgiu, especialmente em projetos com muitos commits no histórico.

Iniciar o bisect

Começa a sessão de investigação:

git bisect start

Marcar o commit ruim (com bug)

Indica que o estado atual tem o bug:

git bisect bad

Marcar um commit bom (sem bug)

Indica um commit do passado onde o bug ainda não existia:

git bisect good abc1234

Ir avançando pela bisection

O Git vai alternando entre commits. Em cada um, teste se o bug existe e marque:

# Se o bug existe neste commit:
git bisect bad

# Se o bug não existe neste commit:
git bisect good

O Git vai afunilando até encontrar o commit exato que causou o problema.

Encerrar o bisect

Quando o Git encontrar o commit culpado, finalize para voltar ao estado normal:

git bisect reset

Exemplo completo

git bisect start
git bisect bad                  # commit atual tem o bug
git bisect good v1.0            # versão 1.0 não tinha o bug

# Git navega automaticamente pelo histórico
# Você testa cada versão e marca:
git bisect good   # ou
git bisect bad

# Git encontra o commit culpado e mostra o hash

# Volta ao estado normal
git bisect reset
💡 Dica: Se tiver um script de teste automatizado, você pode usar git bisect run meu-script.sh e o Git vai fazer toda a bisection sozinho, sem você precisar marcar manualmente.
