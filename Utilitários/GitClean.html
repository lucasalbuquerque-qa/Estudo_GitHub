git clean Utilitários

O comando git clean apaga arquivos não rastreados pelo Git da pasta do projeto — aqueles que nunca foram adicionados com git add. Útil para limpar arquivos temporários e resíduos de build.

Quando usar

Use git clean quando quiser remover arquivos soltos que o Git não rastreia, como logs temporários, arquivos gerados por ferramentas ou resíduos de compilação.

Visualizar o que será removido

Antes de apagar, veja quais arquivos seriam removidos com o modo dry-run (-n):

git clean -n

Remover arquivos não rastreados

O -f (force) é obrigatório para executar a limpeza:

git clean -f

Remover também pastas não rastreadas

Por padrão o clean não remove pastas. Use -fd para incluir diretórios:

git clean -fd

Remover arquivos ignorados também

Para limpar até os arquivos listados no .gitignore, use -fx:

git clean -fx

Modo interativo

Para escolher um por um o que remover:

git clean -i

Exemplo completo

# Primeiro veja o que será removido
git clean -n

# Depois execute a limpeza
git clean -fd

# Confirme que está limpo
git status
⚠️ Atenção: O git clean é irreversível — os arquivos removidos não vão para a lixeira. Sempre rode git clean -n antes para conferir o que será apagado.
