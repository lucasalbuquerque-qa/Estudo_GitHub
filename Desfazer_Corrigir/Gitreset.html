git reset Desfazer / Corrigir

O comando git reset move o ponteiro da branch para um commit anterior, podendo desfazer commits locais. Diferente do revert, ele reescreve o histórico.

Quando usar

Use git reset para desfazer commits que ainda não foram enviados ao GitHub. Evite usá-lo após o push para não criar conflitos.

Reset suave — mantém as alterações

Desfaz o commit mas mantém os arquivos modificados no stage:

git reset --soft HEAD~1

Reset misto — mantém os arquivos

Desfaz o commit e tira os arquivos do stage, mas mantém as alterações nos arquivos:

git reset HEAD~1

Reset total — apaga tudo

Desfaz o commit e apaga todas as alterações dos arquivos. Use com cuidado:

git reset --hard HEAD~1

Voltar para um commit específico

Use o hash do commit para voltar a um ponto específico:

git reset --hard abc1234

Exemplo completo

git log --oneline
git reset --soft HEAD~1
git status
💡 Dica: HEAD~1 significa "1 commit atrás do atual". HEAD~3 volta 3 commits. Use --soft quando quiser refazer o commit com mensagem diferente.
