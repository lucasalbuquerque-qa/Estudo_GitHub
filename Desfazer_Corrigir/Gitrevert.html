git revert Desfazer / Corrigir

← Voltar

O comando git revert desfaz um commit criando um novo commit que reverte as alterações. É a forma segura de desfazer mudanças sem apagar o histórico.

Quando usar

Use git revert quando quiser desfazer um commit que já foi enviado para o GitHub, sem reescrever o histórico do projeto.

Encontrar o hash do commit

Primeiro veja o histórico para pegar o código do commit:

git log --oneline

Reverter o commit

Use o hash do commit que quer desfazer:

git revert abc1234

Reverter sem abrir o editor

Para criar o commit de reversão direto, sem abrir o editor de mensagem:

git revert abc1234 --no-edit

Enviar a reversão para o GitHub

Após reverter, envie as alterações:

git push

Exemplo completo

git log --oneline
git revert abc1234 --no-edit
git push
💡 Dica: Prefira git revert ao git reset quando o commit já foi enviado ao GitHub, pois ele não reescreve o histórico.
