git restore Desfazer / Corrigir

O comando git restore restaura arquivos para um estado anterior sem mudar de branch. É o comando moderno para desfazer alterações em arquivos, substituindo o uso do git checkout -- arquivo.

Quando usar

Use git restore quando fizer alterações em um arquivo e quiser desfazê-las, voltando ao estado do último commit ou retirando arquivos do stage.

Desfazer alterações em um arquivo

Volta o arquivo ao estado do último commit, descartando as mudanças locais:

git restore nome-do-arquivo.html

⚠️ Atenção: essa ação é irreversível. As alterações não commitadas serão perdidas.

Desfazer alterações em todos os arquivos

Descarta todas as mudanças não commitadas do projeto:

git restore .

Tirar um arquivo do stage

Remove um arquivo da área de stage sem perder as alterações (desfaz o git add):

git restore --staged nome-do-arquivo.html

Tirar todos os arquivos do stage

Remove todos os arquivos que foram adicionados com git add:

git restore --staged .

Restaurar a partir de um commit específico

Traz a versão de um arquivo de um commit anterior:

git restore --source abc1234 nome-do-arquivo.html

Exemplo completo

# Ver o que está modificado
git status

# Tirar arquivo do stage
git restore --staged index.html

# Desfazer alterações no arquivo
git restore index.html

# Confirmar que voltou ao normal
git status
💡 Dica: Use git restore --staged para desfazer o git add e git restore para desfazer as edições no arquivo. São ações diferentes e independentes!
