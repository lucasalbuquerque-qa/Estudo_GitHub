git remote Iniciar Projeto

O comando git remote gerencia as conexões entre o seu repositório local e repositórios remotos no GitHub. Permite adicionar, remover, renomear e listar essas conexões.

Quando usar

Use git remote quando precisar conectar seu projeto local a um repositório no GitHub, trocar a URL de origem, ou gerenciar múltiplos remotos como no caso de forks.

Listar os remotos existentes

Mostra todos os repositórios remotos conectados ao projeto:

git remote -v

Adicionar um remoto

Conecta o projeto local a um repositório no GitHub:

git remote add origin https://github.com/usuario/repositorio.git

origin é o nome padrão dado ao remoto principal, mas pode ser qualquer nome.

Trocar a URL de um remoto

Atualiza o endereço de um remoto já existente:

git remote set-url origin https://github.com/usuario/novo-repo.git

Renomear um remoto

Muda o nome de uma conexão remota:

git remote rename origin producao

Remover um remoto

Desconecta o projeto de um repositório remoto:

git remote remove origin

Exemplo completo

# Ver remotos atuais
git remote -v

# Adicionar remoto
git remote add origin https://github.com/usuario/repo.git

# Trocar URL (útil para corrigir conta errada)
git remote set-url origin https://usuario@github.com/usuario/repo.git

# Confirmar a troca
git remote -v
💡 Dica: Em projetos com fork, é comum ter dois remotos: origin apontando para o seu fork e upstream apontando para o repositório original.
