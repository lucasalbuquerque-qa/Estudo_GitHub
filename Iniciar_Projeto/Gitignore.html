git ignore Configuração

O arquivo .gitignore diz ao Git quais arquivos e pastas ele deve ignorar e nunca rastrear. Muito usado para não enviar ao GitHub arquivos como senhas, configurações locais, dependências e arquivos gerados automaticamente.

Quando usar

Use o .gitignore para evitar que arquivos desnecessários ou sensíveis sejam enviados ao GitHub, como a pasta node_modules, arquivos .env com senhas, ou arquivos temporários do sistema.

Criar o arquivo .gitignore

Na raiz do projeto, crie o arquivo pelo terminal:

touch .gitignore

Ou simplesmente crie um novo arquivo no VS Code com o nome .gitignore (com o ponto na frente).

Adicionar arquivos e pastas para ignorar

Abra o .gitignore e escreva o nome do que quer ignorar, um por linha:

# Dependências
node_modules/

# Variáveis de ambiente e senhas
.env
.env.local

# Arquivos do sistema
.DS_Store
Thumbs.db

# Pasta de build
dist/
build/

# Logs
*.log

# Arquivos do editor
.vscode/
.idea/

Entendendo os padrões

Você pode usar padrões para ignorar grupos de arquivos:

# Ignora todos os arquivos .log
*.log

# Ignora uma pasta específica e tudo dentro dela
node_modules/

# Ignora um arquivo específico na raiz
/config.local.js

# Ignora arquivos em qualquer subpasta com esse nome
**/temp/

# Não ignora este arquivo mesmo que o padrão acima pegue
!importante.log

Verificar o que está sendo ignorado

Para confirmar se um arquivo está sendo ignorado pelo Git:

git check-ignore -v nome-do-arquivo

Para listar todos os arquivos ignorados no projeto:

git status --ignored

Parar de rastrear um arquivo já commitado

Se você já commitou um arquivo por engano e agora quer ignorá-lo, primeiro adicione-o ao .gitignore e depois remova do rastreamento:

# Remove do rastreamento mas mantém o arquivo localmente
git rm --cached nome-do-arquivo

# Para uma pasta inteira
git rm --cached -r nome-da-pasta/

# Depois commite a remoção
git add .
git commit -m "Remove arquivo do rastreamento"

Exemplo completo — projeto Node.js

# Cria o .gitignore
touch .gitignore

# Conteúdo do .gitignore:
node_modules/
.env
dist/
*.log
.DS_Store

# Verifica se está funcionando
git status
git check-ignore -v node_modules
💡 Dica: O site gitignore.io gera automaticamente um .gitignore completo para sua linguagem ou framework. Basta digitar "Node", "Python", "React" e ele monta o arquivo para você.
