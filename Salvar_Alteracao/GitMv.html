git mv Salvar Alterações

O comando git mv renomeia ou move arquivos e pastas já rastreados pelo Git, registrando a mudança automaticamente no stage, pronto para o próximo commit.

Quando usar

Use git mv em vez de renomear arquivos manualmente pelo sistema operacional. Quando você renomeia fora do Git, ele interpreta como um arquivo deletado e outro criado — o git mv preserva o histórico corretamente.

Renomear um arquivo

Muda o nome do arquivo e já adiciona a mudança ao stage:

git mv nome-antigo.html nome-novo.html

Mover um arquivo para outra pasta

Move o arquivo para um diretório diferente:

git mv arquivo.html pasta/arquivo.html

Mover e renomear ao mesmo tempo

Você pode mover e já renomear em um único comando:

git mv nome-antigo.html nova-pasta/nome-novo.html

Verificar e commitar

Após o mv, a mudança já está no stage. Confirme e commite:

git status
git commit -m "Renomeia arquivo X para Y"

Exemplo completo

git mv index.html home.html
git status
git commit -m "Renomeia index.html para home.html"
💡 Dica: Se você já renomeou o arquivo fora do Git, pode corrigir com git add -A que detecta renomeações automaticamente antes do commit.
