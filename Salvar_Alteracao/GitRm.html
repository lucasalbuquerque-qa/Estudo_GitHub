git rm Salvar Alterações

O comando git rm remove arquivos do projeto e do rastreamento do Git ao mesmo tempo, registrando a remoção no stage para o próximo commit.

Quando usar

Use git rm quando quiser deletar um arquivo do projeto e parar de rastreá-lo. Se você deletar pelo explorador de arquivos, o Git detecta a remoção mas você ainda precisa fazer o add e commit.

Remover um arquivo do projeto e do Git

Deleta o arquivo localmente e registra a remoção no stage:

git rm nome-do-arquivo.html

Remover uma pasta inteira

Use o -r para remover pastas recursivamente:

git rm -r nome-da-pasta/

Parar de rastrear sem deletar o arquivo

Remove do rastreamento do Git mas mantém o arquivo no seu computador. Muito útil para arquivos que foram commitados por engano:

git rm --cached nome-do-arquivo.html

# Para uma pasta inteira:
git rm --cached -r nome-da-pasta/

Commitar a remoção

Após o rm, a remoção já está no stage. Confirme e commite:

git status
git commit -m "Remove arquivo desnecessário"

Exemplo completo

# Remover arquivo commitado por engano (mantém local)
git rm --cached senha.txt
echo "senha.txt" >> .gitignore
git add .gitignore
git commit -m "Remove senha.txt do rastreamento"
💡 Dica: O uso mais comum do git rm --cached é para remover do GitHub arquivos que foram commitados antes de existir o .gitignore, como a pasta node_modules/.
