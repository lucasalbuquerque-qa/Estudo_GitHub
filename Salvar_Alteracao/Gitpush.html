git push Salvar Alterações

← Voltar

O comando git push envia os commits locais para o repositório remoto no GitHub, tornando as alterações visíveis para outros colaboradores.

Quando usar

Use git push depois de fazer os commits locais e quiser enviá-los para o GitHub.

Enviar para a branch atual

Envia os commits da branch atual para o repositório remoto:

git push

Enviar para uma branch específica

Indica o repositório remoto e a branch de destino:

git push origin main

Primeiro push de uma branch nova

Quando a branch ainda não existe no GitHub, use o -u para vincular:

git push -u origin nome-da-branch

Verificar após o push

Confirme que não há mais commits pendentes:

git status

Exemplo completo

git add .
git commit -m "Adiciona nova funcionalidade"
git push origin main
git status
💡 Dica: Se o push for negado por falta de permissão, verifique se a URL do remote contém o usuário correto com git remote -v.
