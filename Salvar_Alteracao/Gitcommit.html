git commit Salvar Alterações

← Voltar

O comando git commit salva as alterações do stage no histórico do repositório, criando um ponto de controle com uma mensagem descritiva.

Quando usar

Use git commit depois do git add, quando quiser registrar as alterações feitas com uma mensagem explicando o que foi feito.

Fazer um commit com mensagem

O -m permite escrever a mensagem direto no comando:

git commit -m "Mensagem descrevendo o que foi feito"

Adicionar e commitar ao mesmo tempo

O -a adiciona automaticamente os arquivos já rastreados e commita:

git commit -a -m "Mensagem do commit"

Corrigir a mensagem do último commit

Se errou a mensagem, corrija antes de fazer o push:

git commit --amend -m "Mensagem corrigida"

Ver o commit criado

Confirme que o commit foi registrado:

git log --oneline

Exemplo completo

git add .
git commit -m "Adiciona página de contato"
git log --oneline
💡 Dica: Escreva mensagens claras e no imperativo: "Adiciona login", "Corrige bug no formulário", "Atualiza README".
