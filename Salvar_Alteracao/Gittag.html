git tag Salvar Alterações

O comando git tag marca um ponto específico no histórico do projeto como uma versão importante, como v1.0 ou v2.3.1. Tags são usadas para marcar releases e versões estáveis.

Quando usar

Use git tag quando quiser marcar uma versão do projeto para lançamento, como ao publicar uma nova versão de um sistema ou biblioteca.

Listar todas as tags

Mostra todas as tags criadas no repositório:

git tag

Criar uma tag simples

Cria uma tag leve no commit atual:

git tag v1.0

Criar uma tag anotada

Tag anotada inclui mensagem, autor e data — recomendada para releases:

git tag -a v1.0 -m "Versão 1.0 — lançamento inicial"

Enviar tags para o GitHub

Tags não são enviadas automaticamente com o push. Envie separado:

# Enviar uma tag específica
git push origin v1.0

# Enviar todas as tags de uma vez
git push origin --tags

Deletar uma tag

Remove a tag localmente e no repositório remoto:

# Deletar local
git tag -d v1.0

# Deletar no GitHub
git push origin --delete v1.0

Exemplo completo

git tag
git tag -a v1.0 -m "Versão 1.0 — lançamento inicial"
git tag
git push origin --tags
git log --oneline
💡 Dica: Use o padrão vMAJOR.MINOR.PATCH para versionar. Exemplo: v1.0.0 para lançamento, v1.0.1 para correção de bug, v1.1.0 para nova funcionalidade.
