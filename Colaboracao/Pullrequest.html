pull request Colaboração

O Pull Request (PR) é uma solicitação para que o dono de um repositório revise e aceite as suas alterações. É a forma oficial de contribuir com projetos no GitHub.

Quando usar

Use o Pull Request quando terminar uma funcionalidade em uma branch e quiser integrar ao projeto principal, seja no seu próprio repositório ou em um repositório de outra pessoa via fork.

Preparar a branch com suas alterações

Certifique-se de que suas alterações estão commitadas e enviadas para o GitHub:

git checkout -b minha-feature
git add .
git commit -m "Adiciona nova funcionalidade"
git push origin minha-feature

Abrir o Pull Request no GitHub

Após o push, acesse o repositório no GitHub. Vai aparecer um botão amarelo "Compare & pull request". Clique nele, escreva um título e descrição explicando o que foi feito e clique em Create pull request.

Aguardar a revisão

O dono do repositório vai revisar o código. Ele pode:

# Aprovar e fazer o merge — suas alterações entram no projeto
# Pedir mudanças — você edita, commita e o PR atualiza automaticamente
# Fechar sem aceitar — o PR é recusado

Atualizar o PR se pedido mudanças

Se o revisor pedir ajustes, faça na mesma branch e envie de novo:

git add .
git commit -m "Ajusta conforme revisão"
git push origin minha-feature

Exemplo completo

# 1. Cria e envia a branch
git checkout -b corrige-bug-login
git add .
git commit -m "Corrige bug no formulário de login"
git push origin corrige-bug-login

# 2. Abre o PR no GitHub pelo botão "Compare & pull request"

# 3. Se pedirem ajuste:
git add .
git commit -m "Corrige indentação conforme revisão"
git push origin corrige-bug-login
💡 Dica: Escreva uma boa descrição no PR explicando o que foi feito e por quê. Isso facilita a revisão e aumenta a chance do PR ser aceito mais rápido.
