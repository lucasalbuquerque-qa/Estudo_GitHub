fork Colaboração

O fork é uma cópia de um repositório de outra pessoa para a sua conta no GitHub. Com ele você pode modificar o projeto livremente sem afetar o original, e depois propor suas alterações através de um Pull Request.

Quando usar

Use o fork quando quiser contribuir com um projeto open source ou usar o código de outra pessoa como base para o seu próprio projeto.

Fazer o fork no GitHub

Acesse o repositório no GitHub e clique no botão Fork no canto superior direito. O GitHub vai criar uma cópia do repositório na sua conta.

Clonar o fork para sua máquina

Depois do fork, clone o repositório da sua conta (não o original):

git clone https://github.com/SEU-USUARIO/nome-do-repo.git
cd nome-do-repo

Conectar ao repositório original

Adicione o repositório original como remote para receber atualizações futuras:

git remote add upstream https://github.com/DONO-ORIGINAL/nome-do-repo.git
git remote -v

Manter o fork atualizado

Para trazer as novidades do repositório original para o seu fork:

git fetch upstream
git checkout main
git merge upstream/main
git push origin main

Exemplo completo

# 1. Fork feito no GitHub pelo botão Fork

# 2. Clona o fork
git clone https://github.com/SEU-USUARIO/projeto.git
cd projeto

# 3. Conecta ao original
git remote add upstream https://github.com/ORIGINAL/projeto.git

# 4. Cria branch para sua alteração
git checkout -b minha-contribuicao

# 5. Faz as mudanças e commita
git add .
git commit -m "Minha contribuição"
git push origin minha-contribuicao
💡 Dica: Após o push, o GitHub vai sugerir automaticamente abrir um Pull Request para o repositório original. É assim que sua contribuição chega ao dono do projeto!
