git merge Branch (Ramos)

← Voltar

O comando git merge une o conteúdo de uma branch com outra, combinando as alterações feitas em paralelo em um único histórico.

Quando usar

Use git merge quando terminar o desenvolvimento em uma branch e quiser trazer essas alterações para a branch principal.

Ir para a branch que receberá as alterações

Normalmente o merge é feito na branch main:

git checkout main

Mesclar a branch desejada

Traz as alterações da outra branch para a atual:

git merge nome-da-branch

Resolver conflitos se houver

Se o Git avisar conflito, abra os arquivos, resolva manualmente e execute:

git add .
git commit -m "Resolve conflito no merge"

Ver o resultado

Confirme que o merge foi concluído:

git log --oneline --graph

Exemplo completo

git checkout main
git merge minha-feature
git log --oneline --graph
💡 Dica: Após o merge, você pode deletar a branch com git branch -d nome-da-branch para manter o repositório organizado.
